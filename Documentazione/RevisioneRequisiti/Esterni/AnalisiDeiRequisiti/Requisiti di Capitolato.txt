1 CAPITOLATO

	OBBLIGATORI

	-Il sistema deve essere in grado di realizzare diagrammi delle classi
	-deve essere presente un disegnatore per il diagramma delle classi
	-Il sistema deve essere in grado di realizzare diagrammi delle attività
	-deve essere presente un disegnatore per il diagramma delle classi
	-Il sistema deve essere in grado di produrre codice sorgente in linguaggio Java
	-Il sistema deve essere in grado di produrre codice sorgente in linguaggio Javascript
	-Il codice generato deve essere quanto più completo possibilie ma senza errori
	-Il sistema deve essere in grado di indicare quando non è possibile realizzare codice sorgente
	-deve essere presente almeno un coordinatore dche governa la traduzione in codice dei diagrammi disegnati
	-Il sistema deve essere in grado di associare un diagramma delle attività a un paricolare metodo di una classe rappresentata nel diagramma delle classi
	-Il sistema deve rendere disponibili le informazioni specificate nel diagramma associato (es. attributi e metodi della classe mentre viene disegnato il diagramma di attività per un metodo)

	
	OPZIONALI 
	Desiderabili
	-Il sistema deve essere in grado di realizzare un diagramma flowchart
	-Realizzare un gioco Hex utilizzando l'editor creato
	-L'applicazione dovrà essere opensource
	-Introdurre nei disegnatori la possibilità di creare dei "Pattern"
	
	Facoltativi
	-Il sistema deve essere in grado di realizzare un sequence diagram
	-Il sistema deve essere in grado di realizzare un diagramma dei package
	-Estendere o modificare i diagrammi già presenti al fine di migliorare la qualità del codice
	-Creare un sistema che permetta la realizzazione di piccole modifiche al sorgente generato


	OBBLIGATORI MA PROBABILMENTE INTERNI 

	-Dovrà essere fornito un manuale d'utilizzo dell'applicazione
	-Dovrà essere fornito un manuale per estendere l'applicazione
